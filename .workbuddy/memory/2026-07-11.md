## 第三次修复 (v0.1.2 -> v0.1.3) - 关键坐标系 bug
- 根因: v0.1.2 翻转轨道方向时, mesh 的局部坐标错误地包含了 zStart
  - mesh 是 group 的子对象, position 应为局部坐标
  - 但代码写的是 gravel.position.set(0, -0.1, zStart - CHUNK_LENGTH/2)
  - 导致世界坐标 = group.position.z + mesh.position.z = zStart + (zStart - offset) = 2*zStart - offset
  - 整个场景几何位置全乱, 摄像机看到错误的位置
- 同时碰撞检测/金币收集用了 mesh.position.z (局部坐标) 而非世界坐标
  - 应该是 mesh.position.z + chunk.group.position.z
- 修复: createChunk 中所有 mesh 使用纯局部坐标 (z: 0 到 -CHUNK_LENGTH)
  - generateChunkContent 使用局部 z 坐标
  - checkCollisions/collectCoins 正确计算 worldZ = mesh.position.z + chunk.group.position.z
  - startGame 重置 playerHolder 位置
  - 删除 obstacle 对象中未使用的 lane/z 字段
